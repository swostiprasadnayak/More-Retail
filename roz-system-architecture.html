Offline Architecture + Feedback Loops

How Roz works on 2G connections in Tier-2 India, and how every manager action makes the AI smarter.

Offline Architecture — designed for 2G on mid-range Android
☁️
Pre-Computed Server-Side
All ML recommendations computed on AWS, then cached on-device. Phone never runs inference — just displays results.
📤
Offline Queue
Any action taken without connectivity is queued locally and synced when connection returns. No action is ever lost.
💾
Critical Data Cached
Today's order draft, near-expiry list, roster, last-known inventory levels — always available even offline.
⚠️
Graceful Degradation
If server unreachable, Roz shows last-synced state with "stale data" badge. Never a blank screen. Never blocks the manager.
Feedback Loops — every action teaches Roz
📊
Adjusts order qty before approving
Demand model learns her local intuition — festival weeks, vendor quirks, neighborhood patterns no data captures.
🥭
Taps "early markdown" on a mango batch
Per-vendor shelf-life default adjusts. Vendor A's mangoes = 3 days, Vendor B's = 5 days. Learned over time.
📸
Rejects a crate with photo
Vendor quality score accumulates. Same vendor can't send bad produce 3 weeks in a row without it surfacing.
🎉
Tags a local event / festival
Festival calendar grows store-by-store. Next year, Roz knows about the local mango festival before it happens.
🔕
Dismisses a peak alert
Threshold calibrates — was it a false positive? Roz learns to reduce noise for this store's specific patterns.
💰
Overrides markdown %
Clearance model adjusts per category. If 20% always undersells, model bumps to 25% next time for that SKU.
Every manager action flows back into the model.
Roz gets more accurate with every store, every day, without anyone explicitly "training" it.
