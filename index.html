"Daily" in Hindi — an AI ops assistant for the more. store manager

Your 2 Screens — What to Design

Detailed spec for each screen: what it shows, what the user does, and exactly what UI elements to include.

Screen 01 · 12:00 PM Use Case

Smart Order — Morning Order Review

Radhika opens Roz after breakfast. In 90 seconds she reviews today's AI-generated order for fresh produce, adjusts one item, and sends it to the DC. Done. The old process took 60 minutes.

  • Header: "Aaj ka order" + store name + date. Simple orange nav bar.
  • Summary bar: Total items · Total qty · Estimated ₹ value · Risk flag (e.g. "3 items low confidence")
  • Order list: Each item = name in Hindi + English, AI qty rec, current stock, yesterday sold, trend arrow (↑↓), confidence pill (High/Med/Low). Stepper to adjust qty.
  • Confidence explained: Low confidence items expand to show why ("festival tomorrow — uncertain demand")
  • Action bar (sticky bottom): "Approve All" (primary orange CTA) + "Save draft" secondary
  • Status strip: "Last order sent 2 days ago — wastage 4.2 kg" — feedback loop visible
Screen 02 · 8:00 PM Use Case

Daily Briefing — Auto-Generated HQ Report

Radhika opens Roz at store close. Her daily HQ report is already written — pulled from POS and inventory. She reads it, adds one voice note, taps send. 2 minutes. Not 60.

  • Header: "Aaj ka report" + auto-generated timestamp. "Ready to send" status chip.
  • KPI row (3 numbers): Today's sales vs target · Wastage kg · In-stock % — color coded green/red
  • Anomaly callouts: 1–3 flagged items ("Tomato sales 40% below yesterday — possible quality issue"). Each has a "Mark resolved" tap.
  • Summary text: 3-line auto-written summary in Hindi. Editable. Manager can tap to add voice note.
  • Expiry cleared: List of items marked down today + qty recovered vs written off
  • Action bar (sticky bottom): "Send to Area Manager" (WhatsApp share) + "Edit" secondary

Blueprint Insight 01
The data already exists
Every decision Radhika makes manually has the supporting data sitting in Oracle ERP or the POS system. Roz doesn't collect new data — it surfaces what's already there.
Blueprint Insight 02
AI sits between data and decision
Roz lives in the gap between support systems (ERP/POS) and the manager's frontstage actions. It translates raw data into a single, approvable recommendation.
Blueprint Insight 03
Manager stays in control
Roz recommends — Radhika decides. Every AI action requires a human tap to confirm. This is critical for trust with a frontline manager who has always relied on her own judgment.
